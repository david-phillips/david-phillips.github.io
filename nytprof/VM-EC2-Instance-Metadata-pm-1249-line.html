60=head1 DESCRIPTION
61
62This is an interface to the metadata that is provided to a running
63instance via the http://169.254.169.254/latest URL, as described in
64http://docs.amazonwebservices.com/AWSEC2/latest/UserGuide/index.html?instancedata-data-categories.html.
65
66Each metadata object caches its values, so there is no overhead in
67calling a method repeatedly. Methods return scalar values, lists and
68hashrefs as appropriate.
69
70The methods from this class should only be called within the context
71of a running EC2 instance. Attempts to call outside of this
72context will result in long delays as the module attempts to
73connect to an invalid hostname.
74
75=head1 METHODS
76
77=head2 $meta = VM::EC2::Instance::Metadata->new()
78

80
81=head2 $meta = $instance->metadata()
82
83You can create a new metadata object either using this class's new()
84constructor, or by calling an VM::EC2 object's instance_metadata()
85method, or by calling a VM::EC2::Instance object's metadata () method.
86
87=head2 Methods that return scalar values
88
89The following methods all return single-valued results:
90
91=over 4
92
93=item Image information:
94
95 imageId -- ID of AMI used to launch this instance
96 imageLaunchIndex -- This index's launch index. If four instances
97 were launched by one $image->run_instances()
98 call, they will be numbered from 0 to 3.
99 amiManifestPath -- S3 path to the image
100 imageLocation -- Same as amiManifestPath(), for consistency with
101 VM::EC2::Image
102
103=item Launch and runtime information:
104
105 instanceId -- ID of this instance
106 kernelId -- ID of this instance's kernel.
107 ramdiskId -- This instance's ramdisk ID
108 reservationId -- This instance's reservation ID
109 instanceType -- Machine type, e.g. "m1.small"
110 availabilityZone -- This instance's availability zone.
111 region -- This instance's region.
112 endpoint -- This instance's endpoint.
113 userData -- User data passed at launch time.
114
115=item Network information:
116
117 localHostname -- The instance hostname corresponding to its
118 internal EC2 IP address.
119 privateDnsName -- Same as localHostname(), for consistency with
120 VM::EC2::Instance
121 localIpv4 -- The instance IP address on the internal EC2 network.
122 privateIpAddress -- Same as localIpv4(), for consistency with
123 VM::EC2::Instance.
124 mac -- This instance's MAC (ethernet) address.
125 publicHostname -- This instance's public hostname.
126 dnsName -- Same as publicHostname() for consistency with
127 VM::EC2::Instance.
128 publicIpv4 -- This instance's public IP address.
129 ipAddress -- Same as publicIpv4() for consistency with
130 VM::EC2::Instance.
131
132=item IAM information
133
134These routines return information about the instance's IAM role, if
135any. These calls also provide a temporary security credentials for
136making EC2 calls, as described here:
137http://docs.amazonwebservices.com/AWSEC2/latest/UserGuide/UsingIAM.html.
138
139Note that these routines require installation of the perl JSON module, and will
140cause a fatal error if this module cannot be loaded.
141
142 iam_info -- Returns a hash containing the fields 'LastUpdated',
143 'InstanceProfileArn', and 'InstanceProfileId'. These
144 provide information about the instance's IAM role.
145 iam_role -- Returns the IAM role name for the currently running
146 instance.
147 iam_credentials -- Returns a VM::EC2::Security::Credentials object that can
148 be passed to VM::EC2->new(-security_token=>$credentials).
149
150=item Unknown information:
151
152 profile -- An undocumented field that contains the virtualization
153 type in the form "default-paravirtual".
154 instanceAction -- Undocumented metadata field named "instance-action"
155
156=back
157
158=head2 Methods that return lists
159
160The following methods all return lists.
161
162=over 4
163
164=item Image information
165
166 ancestorAmiIds -- List of AMIs from which the current one was derived
167 imageAncestorIds -- Same as ancestorAmiIds() but easier to read.
168 productCodes -- List of product codes applying to the image from which

170
171=item Launch and runtime information
172
173 securityGroups -- List of security groups to which this instance is assigned.
174 For non-VPC instances, this will be the security group
175 name. For VPC instances, this will be the security group ID.
176 publicKeys -- List of public key pair names attached to this instance.
177
178=back
179
180=head2 Methods that return a hashref
181
182The following methods return a hashref for representing complex data structures:
183
184=over 4
185
186=item $devices = $meta->blockDeviceMapping
187
188This returns a hashref in which the keys are the names of instance
189block devices, such as "/dev/sda1", and the values are the EC2 virtual
190machine names. For example:
191
192 x $meta->blockDeviceMapping
193 0 HASH(0x9b4f2f8)
194 '/dev/sda1' => 'root'
195 '/dev/sda2' => 'ephemeral0'
196 '/dev/sdg' => 'ebs1'
197 '/dev/sdh' => 'ebs9'
198 '/dev/sdi' => 'ebs10'
199 'sda3' => 'swap'
200
201For reasons that are not entirely clear, the swap device is reported
202as "sda3" rather than as "/dev/sda3".
203
204=item $interfaces = $meta->interfaces
205
206Returns a mapping of all virtual ethernet devices owned by this
207instance. This is primarily useful for VPC instances, which can have
208more than one device. The hash keys are the MAC addresses of each
209ethernet device, and the values are hashes that have the following
210keys:
211
212 mac
213 localHostname
214 localIpv4s (an array ref)
215 publicIpv4s (an array ref)
216 securityGroupIds (an array ref)
217 subnetId
218 subnetIpv4CidrBlock
219 vpcId
220 vpcIpv4CidrBlock
221
222For example:
223
224 D
225 x $meta->interfaces
226 0 HASH(0x9b4f518)
227 '12:31:38:01:b8:97' => HASH(0x9eaa090)
228 'localHostname' => 'domU-12-31-38-01-B8-97.compute-1.internal'
229 'localIpv4s' => ARRAY(0x9b4f8a8)
230 0 '10.253.191.101'
231 'mac' => '12:31:38:01:b8:97'
232 'publicIpv4s' => ARRAY(0x9ea9e40)
233 0 '184.73.241.210'
234 'securityGroupIds' => ARRAY(0x9eaa490)
235 empty array
236 'subnetId' => undef
237 'subnetIpv4CidrBlock' => undef
238 'vpcId' => undef
239 'vpcIpv4CidrBlock' => undef
240
241=back
