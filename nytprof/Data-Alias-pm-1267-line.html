20=head1 NAME
21
22Data::Alias - Comprehensive set of aliasing operations
23
24=head1 SYNOPSIS
25
26 use Data::Alias;
27
28 alias {
29 # aliasing instead of copying whenever possible
30 };
31
32 alias $x = $y; # alias $x to $y
33 alias @x = @y; # alias @x to @y
34 alias $x[0] = $y; # similar for array and hash elements
35 alias push @x, $y; # push alias to $y onto @x
36 $x = alias [ $y, $z ]; # construct array of aliases
37 alias my ($x, $y) = @_; # named aliases to arguments
38 alias { ($x, $y) = ($y, $x) }; # swap $x and $y
39 alias { my @t = @x; @x = @y; @y = @t }; # swap @x and @y
40
41 use Data::Alias qw/ alias copy /;
42
43 alias { copy $x = $y }; # force copying inside alias-BLOCK
44
45 use Data::Alias qw/ deref /;
46
47 my @refs = (\$x, \@y, \%z);
48 foo(deref @refs) # same as foo($x, @y, %z)

50=head1 DESCRIPTION
51
52Aliasing is the phenomenon where two different expressions actually refer to
53the same thing. Modifying one will modify the other, and if you take a
54reference to both, the two values are the same.
55
56Aliasing occurs in Perl for example in for-loops and sub-calls:
57
58 for $var ($x) {
59 # here $var is an alias to $x
60 }
61
62 foo($y);
63 sub foo {
64 # here $_[0] is an alias to $y
65 }
66
67Data::Alias is a module that allows you to apply "aliasing semantics" to a
68section of code, causing aliases to be made wherever Perl would normally make
69copies instead. You can use this to improve efficiency and readability, when
70compared to using references.
71
72The exact details of aliasing semantics are below under L</DETAILS>.
73
74=head1 SYNTAX
75
76=head2 alias I<EXPR> | alias I<BLOCK>
77
78Exported by default.
79
80Enables aliasing semantics within the expression or block. Returns an alias
81to the expression, or the block's return value.
82
83C<alias> is context-transparent, meaning that whichever context it is placed in
84(list, scalar, void), the expression/block is evaluated in the same context.
85
86=head2 copy I<EXPR> | copy I<BLOCK>
87
88Restores normal (copying) semantics within the expression or block, and
89makes a copy of the result value (unless in void context).
90
91Like C<alias>, C<copy> is context-transparent.
92
93=head2 deref I<LIST>
94
95Accepts a list of references to scalars, arrays, or hashes. Applies the
96applicable dereferencing operator to each. This means that:
97
98 deref $scalarref, $arrayref, $hashref
99
100behaves like:
101
102 $$scalarref, @$arrayref, %$hashref
103
104Where an array or hash reference is given, the returned list does not
105include the array or hash as an lvalue; the array/hash is expanded and
106the list includes its elements. Scalars, including the elements of an
107array/hash, I<are> treated as lvalues, and can be enreferenced using
108the C<\> operator or aliased to using the C<alias> operator. This is
109slightly different from what you'd get using the built-in dereference
110operators: C<@$arrayref> references the array as an lvalue, so C<\>
111or C<alias> can operate on the array itself rather than just its elements.
112
113=head1 EXAMPLES
114
115A common usage of aliasing is to make an abbreviation for an expression, to
116avoid having to repeat that (possibly verbose or ugly) expression over and
117over:
118
119 alias my $fi = $self->{FrobnitzIndex};
120 $fi = $fi > 0 ? $fi - $adj : $fi + $adj;
121
122 sub rc4 {
123 alias my ($i, $j, $S) = @_;
124 my $a = $S->[($i += 1) &= 255];
125 my $b = $S->[($j += $S->[$i]) &= 255];
126 $S->[(($S->[$j] = $a) + ($S->[$i] = $b)) & 255]
127 }
128
129In the second example, the rc4 function updates its first two arguments (two
130state values) in addition to returning a value.
131
132Aliasing can also be used to avoid copying big strings. This example would
133work fine without C<alias> but would be much slower when passed a big string:
134
135 sub middlesection ($) {
136 alias my $s = shift;
137 substr $s, length($s)/4, length($s)/2
138 }
139
140You can also apply aliasing semantics to an entire block. Here this is used to
141swap two arrays in O(1) time:
142
143 alias {
144 my @temp = @x;
145 @x = @y;
146 @y = @temp;
147 };
148
149The C<copy> function is typically used to temporarily reinstate normal
150semantics, but can also be used to explicitly copy a value when perl would
151normally not do so:
152
153 my $ref = \copy $x;
154
155=head1 DETAILS
156
157This section describes exactly what the aliasing semantics are of operations.
158Anything not listed below has unaltered behaviour.
159
160=over 4
161
162=item scalar assignment to variable or element.
163
164Makes the left-side of the assignment an alias to the right-side expression,
165which can be anything.
166
167 alias my $lexvar = $foo;
168 alias $pkgvar = $foo;
169 alias $array[$i] = $foo;
170 alias $hash{$k} = $foo;
171
172An attempt to do alias-assignment to an element of a tied (or "magical") array
173or hash will result in a "Can't put alias into tied array/hash" error.
174
175=item scalar assignment to dereference
176
177If $ref is a reference or undef, this simply does C<$ref = \$foo>. Otherwise,
178the indicated package variable (via glob or symbolic reference) is made an
179alias to the right-side expression.
180
181 alias $$ref = $foo;
182
183=item scalar assignment to glob
184
185Works mostly the same as normal glob-assignment, however it does not set the
186import-flag. (If you don't know what this means, you probably don't care)
187
188 alias *glob = $reference;
189
190=item scalar assignment to anything else
191
192Not supported.
193
194 alias substr(...) = $foo; # ERROR!
195 alias lvalsub() = $foo; # ERROR!
196
197=item conditional scalar assignment
198
199Here C<$var> (and C<$var2>) are aliased to C<$foo> if the applicable condition
200is satisfied. C<$bool> and C<$foo> can be any expression. C<$var> and
201C<$var2> can be anything that is valid on the left-side of an alias-assignment.
202
203 alias $bool ? $var : $var2 = $foo;
204 alias $var &&= $foo;
205 alias $var ||= $foo;
206 alias $var //= $foo; # (perl 5.9.x or later)
207
208=item whole aggregate assignment from whole aggregate
209
210This occurs where the expressions on both sides of the assignment operator
211are purely complete arrays or hashes.
212The entire aggregate is aliased, not merely the contents.
213This means for example that C<\@lexarray == \@foo>.
214
215 alias my @lexarray = @foo;
216 alias my %lexhash = %foo;
217 alias @pkgarray = @foo;
218 alias %pkghash = %foo;
219
220Making the left-side a dereference is also supported:
221
222 alias @$ref = @foo;
223 alias %$ref = %foo;
224
225and analogously to assignment to scalar dereference, these will change C<$ref>
226to reference the aggregate, if C<$ref> was undef or already a reference. If
227C<$ref> is a string or glob, the corresponding package variable is aliased.
228
229Anything more complex than a whole-aggregate expression on either side,
230even just enclosing the aggregate expression in parentheses, will prevent
231the assignment qualifying for this category. It will instead go into
232one of the following two categories. Parenthesisation is the recommended
233way to avoid whole-aggregate aliasing where it is unwanted. If you want
234to merely replace the contents of the left-side aggregate with aliases
235to the contents of the right-side aggregate, parenthesise the left side.
236
237=item whole aggregate assignment from list
238
239If the left-side expression is purely a complete array or hash,
240and the right-side expression is not purely a matching aggregate, then a new
241aggregate is implicitly constructed. This means:
242
243 alias my @lexfoo = (@foo);
244 alias my @array = ($x, $y, $z);
245 alias my %hash = (x => $x, y => $y);
246
247is translated to:
248
249 alias my @lexfoo = @{ [@foo] };
250 alias my @array = @{ [$x, $y, $z] };
251 alias my %hash = %{ {x => $x, y => $y} };
252
253If you want to merely replace the contents of the aggregate with aliases to the
254contents of another aggregate, rather than create a new aggregate, you can
255force list-assignment by parenthesizing the left side, see below.
256
257=item list assignment
258
259List assignment is any assignment where the left-side is an array-slice,
260hash-slice, or list in parentheses. This behaves essentially like many scalar
261assignments in parallel.
262
263 alias my (@array) = ($x, $y, $z);
264 alias my (%hash) = (x => $x, y => $y);
265 alias my ($x, $y, @rest) = @_;
266 alias @x[0, 1] = @x[1, 0];
267
268Any scalars that appear on the left side must be valid targets for scalar
269assignment. When an array or hash appears on the left side, normally as the
270last item, its contents are replaced by the list of all remaining right-side
271elements. C<undef> can also appear on the left side to skip one corresponding
272item in the right-side list.
273
274Beware when putting a parenthesised list on the left side. Just like Perl
275parses C<print (1+2)*10> as C<(print(1+2))*10>, it would parse C<alias ($x, $y)
276= ($y, $x)> as C<(alias($x, $y)) = ($y, $x)> which does not do any aliasing,
277and results in the "Useless use of alias" warning, if warnings are enabled.
278
279To circumvent this issue, you can either one of the following:
280
281 alias +($x, $y) = ($y, $x);
282 alias { ($x, $y) = ($y, $x) };
283
284=item Anonymous aggregate constructors
285
286Return a reference to a new anonymous array or hash, populated with aliases.
287This means that for example C<\$hashref-E<gt>{x} == \$x>.
288
289 my $arrayref = alias [$x, $y, $z];
290 my $hashref = alias {x => $x, y => $y};
291
292Note that this also works:
293
294 alias my $arrayref = [$x, $y, $z];
295 alias my $hashref = {x => $x, y => $y};
296
297but this makes the lhs an alias to the temporary, and therefore read-only,
298reference made by C<[]> or C<{}>. Therefore later attempts to assign to
299C<$arrayref> or C<$hashref> results in an error. The anonymous aggregate that
300is referenced behaves the same in both cases obviously.
301
302=item Array insertions
303
304These work as usual, except the inserted elements are aliases.
305
306 alias push @array, $foo;
307 alias unshift @array, $foo;
308 alias splice @array, 1, 2, $foo;
309
310An attempt to do any of these on tied (or "magical") array will result in a
311"Can't push/unshift/splice alias onto tied array" error.
312
313=item Returning an alias
314
315Returns aliases from the current C<sub> or C<eval>. Normally this only
316happens for lvalue subs, but C<alias return> can be used in any sub.
317Lvalue subs only work for scalar return values, but C<alias return>
318can handle a list of return values.
319
320A sub call will very often copy the return value(s) immediately after
321they have been returned. C<alias return> can't prevent that. To pass
322an alias through a sub return and into something else, the call site
323must process the return value using an aliasing operation, or at least a
324non-copying one. For example, ordinary assignment with the sub call on
325the right hand side will copy, but if the call site is in the scope of an
326C<alias> pragma then the assignment will instead alias the return value.
327
328When alias-returning a list of values from a subroutine, each individual
329value in the list is aliased. The list as a whole is not aliasable;
330it is not an array. At the call site, a list of aliases can be captured
331into separate variables or into an array, by an aliasing list assignment.
332
333=item Subroutines and evaluations
334
335Placing a subroutine or C<eval STRING> inside C<alias> causes it to be compiled
336with aliasing semantics entirely. Additionally, the return from such a sub or
337eval, whether explicit using C<return> or implicitly the last statement, will
338be an alias rather than a copy.
339
340 alias { sub foo { $x } };
341
342 my $subref = alias sub { $x };
343
344 my $xref1 = \foo;
345 my $xref2 = \alias eval '$x';
346 my $xref3 = \$subref->();
347
348Explicitly returning an alias can also be done using C<alias return> inside any
349subroutine or evaluation.
350
351 sub foo { alias return $x; }
352 my $xref = \foo;
353
354=item Localization
355
356Use of local inside C<alias> usually behaves the same as local does in general,
357however there is a difference if the variable is tied: in this case, Perl
358doesn't localise the variable at all but instead preserves the tie by saving a
359copy of the current value, and restoring this value at end of scope.
360
361 alias local $_ = $string;
362
363The aliasing semantics of C<local> avoids copying by always localizing the
364variable itself, regardless of whether it is tied.
365
366=back
367
368=head1 IMPLEMENTATION
369
370This module does B<not> use a source filter, and is therefore safe to use
371within eval STRING. Instead, Data::Alias hooks into the Perl parser, and
372replaces operations within the scope of C<alias> by aliasing variants.
373
374For those familiar with perl's internals: it triggers on a ck_rv2cv which
375resolves to the imported C<alias> sub, and does a parser hack to allow the
376C<alias BLOCK> syntax. When the ck_entersub is triggered that corresponds to
377it, the op is marked to be found later. The actual work is done in a peep-hook,
378which processes the marked entersub
379and its children, replacing the pp_addrs with aliasing replacements. The peep
380hook will also take care of any subs defined within the lexical (but not
381dynamical) scope between the ck_rv2cv and the ck_entersub.
382
383=head1 KNOWN ISSUES
384
385=over 4
386
387=item Lexical variables
388
389When aliasing existing lexical variables, the effect is limited in scope to the
390current subroutine and any closures create after the aliasing is done, even if
391the variable itself has wider scope. While partial fixes are possible, it
392cannot be fixed in any reliable or consistent way, and therefore I'm keeping
393the current behaviour.
394
395When aliasing a lexical that was declared outside the current subroutine, a
396compile-time warning is generated "Aliasing of outer lexical variable has
397limited scope" (warnings category "closure").
398
399=back
